1. Claude Codeとは何か 1. What Is Claude Code

Claude Codeは、AnthropicがClaude AIモデルをベースに構築したCLIネイティブのAIエージェントだ。ターミナル（コマンドライン）から起動し、ユーザーの指示に従ってローカルファイルの読み書き、コードの生成・実行、複数ファイルにわたる検索・編集を行う。 Claude Code is Anthropic's CLI-native AI agent built on the Claude AI model. Launched from the terminal, it reads and writes local files, generates and executes code, and performs cross-file search and editing based on user instructions.

これはブラウザで使うClaude.aiとは本質的に異なる。Claude.aiはWebアプリケーションであり、ユーザーがテキストを貼り付けてチャットする「会話型インターフェース」だ。Claude Codeはあなたのファイルシステムにアクセスできるエージェントであり、「指示を受けて自律的に作業を完結させる」ことができる。 This is fundamentally different from Claude.ai in the browser. Claude.ai is a web application — a conversational interface where you paste text and chat. Claude Code is an agent with access to your filesystem, capable of receiving instructions and autonomously completing tasks end-to-end.

Finance Pro の視点Finance Pro Perspective

監査・M&A・FP&A業務の現場で感じる最大の摩擦は「AIに渡せるデータ量の制限」だ。有価証券報告書は100ページを超え、DDのVDRには数百ファイルが詰まっている。ブラウザにコピペできる量ではない。Claude Codeはこの制約を根本から崩す。 The biggest friction point in audit, M&A, and FP&A is the data transfer bottleneck. Securities filings exceed 100 pages; VDRs contain hundreds of files. None of this can be copy-pasted into a browser. Claude Code eliminates this constraint at the root.

また、機密性の高い資料をクラウドサービスにアップロードすることへの社内規制は多くの事務所・会社で存在する。ローカルで処理できるというオプションは、セキュリティポリシーの観点からも重要だ（ただし企業別のポリシー確認は必須）。 Many firms also have internal policies restricting upload of sensitive materials to cloud services. The ability to process locally is therefore significant from a security-policy standpoint — though firm-specific policies must always be verified.

Finance Pro の視点 — どのプランを選ぶかFinance Pro Perspective — Which Plan

個人用途であればClaude.ai Proで十分。 月$20で毎日使っても十分な使用枠がある。大量のファイル処理・長大なDD資料の解析・バッチ自動化を行うなら、API従量課金（Claude Opus 4.6: $15/M input tokens）のほうが経済的になるケースがある。 For individual use, Claude.ai Pro is sufficient. $20/month covers daily usage comfortably. If you're doing heavy file processing, large-scale DD analysis, or batch automation, API pay-per-use (Claude Opus 4.6: $15/M input tokens) may be more economical.

チームで使うならClaude Enterprise（セルフサーブ対応）が現実的。セキュリティポリシー・データ保持設定・SSO対応が整っている。監査法人・M&Aアドバイザリー向けにはZero Data Retention（ZDR）オプションの確認が必須。 For team use, Claude Enterprise (now self-serve) is the practical choice. It includes security policy controls, data retention settings, and SSO. For audit firms and M&A advisories, verify Zero Data Retention (ZDR) options — this is non-negotiable for many engagements.

3. 最初の10分でできること 3. What You Can Do in the First 10 Minutes

起動したら対話的なプロンプトが表示される。日本語で指示できる。以下は Finance Pro が最初に試すべき4つの操作だ。 After launch, an interactive prompt appears. You can instruct in Japanese or English. Here are four operations Finance Pros should try first.

① ローカルファイルの読み込みと要約 ① Load and Summarize a Local File

CLAUDE CODE — 指示例
annual_report_2025.pdf を読み込んで、
・売上高・営業利益・純利益の3期比較
・セグメント別業績のサマリー
・リスク要因のトップ5
を表形式でまとめてください

Claude Codeはファイルパスを認識して内容を取り込み、指定されたフォーマットで出力する。ブラウザにPDFをアップロードしてコピペする工程が完全に消える。 Claude Code recognizes the file path, ingests the content, and outputs in the specified format. The entire browser-upload-and-paste workflow disappears.

② 複数ファイルの横断比較 ② Cross-File Comparison

CLAUDE CODE — 指示例
company_A_FS.xlsx と company_B_FS.xlsx を読み込んで
EBITDA・純有利子負債・D/E比率を比較した表を作成してください
差異のある項目には注釈を加えてください

M&AのComparables分析やセクター横断スクリーニングで威力を発揮する。従来はExcelで手作業だった作業が、指示一発で完成する。 This shines in M&A comparables analysis and sector screening. Tasks that used to require manual Excel work complete with a single instruction.

③ レポートの自動生成 ③ Automated Report Generation

CLAUDE CODE — 指示例
data/ フォルダ内の月次P&Lデータ（CSV）を読み込んで 前年同月比・前月比・予実差異を含む 月次業績レポートのHTMLを生成してください ファイル名は monthly_report_2026_03.html としてください

ファイルを読んで、処理して、新しいファイルとして書き出す。この一連の流れが自然言語の指示だけで完結する。月次ルーティンをこれ一本で自動化できる。 Read files, process them, write output — this entire pipeline completes from a natural language instruction. Monthly reporting routines can be fully automated with this alone.

④ CLAUDE.md で永続指示を設定する ④ Set Persistent Instructions via CLAUDE.md

作業ディレクトリに CLAUDE.md というファイルを作成すると、Claude Codeは毎回の起動時にその内容を読み込む。「いつも日本語で回答すること」「財務数値は百万円単位で表示すること」「当社の会計基準はIFRS」など、繰り返し伝えることを一度書けば永続する。 Create a file named CLAUDE.md in your working directory and Claude Code reads it at every startup. "Always respond in English," "Display financial figures in millions," "Our accounting standard is IFRS" — write it once, persist forever.

CLAUDE.md — 設定例（Finance Pro向け）
# Finance Assistant Config ## 基本設定 - 回答は必ず日本語 - 財務数値は百万円単位（小数点1位まで） - 会計基準: IFRS（特に指定がない場合） - 計算結果は必ず根拠を併記すること ## 業務コンテキスト - このフォルダは M&A デューデリジェンス案件用 - 対象会社: [対象会社名] - 基準日: 2025年12月31日 - 通貨: JPY（海外取引はUSD、換算レート:150円） ## 出力ルール - 不確かな情報には [要確認] タグを付けること - 機密情報のスクリーンショットは作成しないこと - 最終成果物はoutput/フォルダに保存すること
Finance Pro の視点 — CLAUDE.mdの実務価値Finance Pro Perspective — CLAUDE.md Value

毎回「IFRSで解釈してください」「百万円単位で」と伝えるのは非効率だ。案件ごとにCLAUDE.mdを作ることで、AIに「この案件の文脈」を覚えさせ続けられる。M&A案件フォルダ・監査クライアントフォルダ・FP&A月次フォルダにそれぞれCLAUDE.mdを置くと、誰でも即座に同じコンテキストでAIを使えるチーム運用が実現する。 Repeating "interpret under IFRS" and "in millions" every session is friction. Creating per-engagement CLAUDE.md files keeps the AI permanently contextualized. Place a CLAUDE.md in each M&A deal folder, audit client folder, and FP&A monthly folder — and any team member instantly gets the same AI context without briefing.
